## 🎯 О проекте

AutoLive — это современная платформа для интернет-магазина автозапчастей, которая включает:

- **Публичный сайт** для клиентов с каталогом товаров, корзиной и оформлением заказов
- **Систему бронирования** для записи на техническое обслуживание
- **Админ-панель** для управления товарами, заказами, пользователями и отделами
- **Многопользовательскую систему** с ролями: суперадмин, админ, менеджер, клиент
- **Систему логирования** всех изменений в системе

## ✨ Основные возможности

### Для клиентов

- 🔍 **Поиск и фильтрация товаров** по категориям, брендам, характеристикам
- 🛒 **Корзина покупок** с сохранением в localStorage
- 📦 **Оформление заказов** без регистрации или с авторизацией
- 📅 **Запись на ТО** с выбором даты, времени и отдела
- 📋 **Комплекты ТО** — готовые наборы для обслуживания
- 👤 **Регистрация и авторизация** через SMS-код
- 📱 **Адаптивный дизайн** для всех устройств

### Для администраторов и менеджеров

- 📊 **Управление товарами** с импортом/экспортом, фильтрами, наценками
- 📦 **Управление заказами** с отслеживанием статусов и назначением менеджеров
- 📅 **Управление записями** на техническое обслуживание
- 👥 **Управление пользователями** и отделами
- 📁 **Управление категориями** с фильтрами и сортировкой
- 📈 **Система логирования** всех действий в системе
- 🏢 **Многодепартаментная система** с разделением по отделам

## 🚀 Установка и настройка

### Требования

- Node.js 18+ 
- PostgreSQL
- npm или yarn

### Шаги установки

1. **Клонируйте репозиторий** (если есть) или скачайте проект

2. **Установите зависимости:**
```bash
npm install
```

3. **Настройте переменные окружения:**

Создайте файл `.env` в корне проекта:

```env
# База данных
DB_URL="postgresql://user:password@localhost:5432/autolive_db"

# JWT секрет для токенов
JWT_SECRET="your-secret-key-here"

# Базовый URL (для продакшена)
NEXT_PUBLIC_BASE_URL="http://localhost:3000"
```

4. **Настройте базу данных:**

```bash
# Применить миграции
npm run db:migrate

# Сгенерировать Prisma Client
npm run db:generate

# (Опционально) Заполнить тестовыми данными
npm run seed
```

5. **Запустите сервер разработки:**

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## 🔐 Роли пользователей

- **superadmin** — полный доступ ко всем функциям и отделам
- **admin** — управление своим отделом (товары, заказы, пользователи)
- **manager** — просмотр и управление заказами своего отдела
- **client** — доступ к публичной части сайта, оформление заказов

## 📝 Особенности реализации

### Система загрузки файлов

Все загрузки (акции, товары, категории, аватары) идут в **Vercel Blob** — и локально, и на production.
Нужна одна переменная `BLOB_READ_WRITE_TOKEN` в `.env.local` и в Vercel. Подробно: [VERCEL_BLOB_SETUP.md](./VERCEL_BLOB_SETUP.md).

### Система логирования

Все изменения в системе логируются с полными снапшотами данных:
- Изменения товаров, заказов, пользователей
- История действий администраторов
- Возможность восстановления данных

### Многодепартаментная система

- Каждый отдел имеет свой набор товаров и категорий
- Заказы могут быть привязаны к отделам
- Менеджеры видят только заказы своего отдела
