### 2. Создать новую сущность и страницу "Пункты выдачи"
**Статус:** ✅ Выполнено (февраль 2025). Яндекс.Карты — в задаче «Редактор контактов».

**Описание:** 
Создать новую сущность "Пункты выдачи" и страницу для её управления, которая будет максимально похожа на страницу "Адреса для записей" (`/admin/booking-departments`).

**Что нужно сделать:**
1. Создать новую модель в Prisma для "Пункты выдачи" (PickupPoint или DeliveryPoint)
2. Создать миграцию базы данных для новой таблицы
3. Создать страницу `/admin/pickup-points` (или `/admin/delivery-points`) со списком пунктов выдачи
4. Создать страницу редактирования `/admin/pickup-points/[id]/edit`
5. Создать страницу логов `/admin/pickup-points/logs` и `/admin/pickup-points/[id]/logs`
6. Создать API endpoints:
   - `GET/POST /api/pickup-points` - список и создание
   - `GET/PUT/DELETE /api/pickup-points/[id]` - получение, обновление, удаление
   - `GET /api/pickup-points/logs` - логи всех пунктов выдачи
   - `GET /api/pickup-points/[id]/logs` - логи конкретного пункта выдачи
7. Реализовать компоненты аналогичные компонентам для "Адреса для записей":
   - `AllPickupPointsTable.tsx` - таблица со списком пунктов выдачи
   - `PickupPointFormComponent.tsx` - форма создания/редактирования
   - `AllPickupPointsLogsTable.tsx` - таблица логов всех пунктов выдачи
   - `PickupPointLogsTable.tsx` - таблица логов конкретного пункта выдачи
8. Добавить интеграцию с Яндекс.Картами для указания точек на карте (как в задаче 3)
9. Добавить права доступа и проверки прав аналогично "Адреса для записей"

**Структура должна быть максимально похожа на:**
- `/admin/booking-departments` - список адресов для записей
- `/admin/booking-departments/[id]/edit` - редактирование адреса
- `/admin/booking-departments/logs` - логи всех адресов
- `/admin/booking-departments/[id]/logs` - логи конкретного адреса

**Файлы для создания:**
- `prisma/schema.prisma` - добавить модель PickupPoint
- `src/app/admin/pickup-points/page.tsx` - главная страница со списком
- `src/app/admin/pickup-points/create/page.tsx` - создание нового пункта выдачи
- `src/app/admin/pickup-points/[id]/edit/page.tsx` - редактирование
- `src/app/admin/pickup-points/[id]/logs/page.tsx` - логи конкретного пункта
- `src/app/admin/pickup-points/logs/page.tsx` - логи всех пунктов
- `src/app/admin/pickup-points/local_components/allPickupPoints/AllPickupPointsTable.tsx`
- `src/app/admin/pickup-points/local_components/pickupPoint/PickupPointFormComponent.tsx`
- `src/app/admin/pickup-points/local_components/allPickupPointsLogs/AllPickupPointsLogsTable.tsx`
- `src/app/admin/pickup-points/local_components/pickupPointLogs/PickupPointLogsTable.tsx`
- `src/app/api/pickup-points/route.ts` - API для списка и создания
- `src/app/api/pickup-points/[id]/route.ts` - API для получения, обновления, удаления
- `src/app/api/pickup-points/logs/route.ts` - API для логов всех пунктов
- `src/app/api/pickup-points/[id]/logs/route.ts` - API для логов конкретного пункта

Эта сущность будет использоваться в редакторе контактов (задача 4) для отображения пунктов выдачи рядом с "Адреса для записей".

## 📝 Заметки

- Все заявки на бронирование обрабатываются менеджерами вручную
- Менеджеры ведут учёт заявок НЕ на сайте
- Клиент выбирает отдел, заявка попадает в общие заявки отдела без назначенного менеджера
- Менеджеры потом берут заявки себе и присваивают их себе
