### 4. Редактор контактов (страница «Контакты»)
**Статус:** ✅ Выполнено (март 2026)

**Описание:**  
Отдельный раздел контента для публичной страницы «Контакты»: собственные контактные адреса с координатами, настройки карты, отображение на сайте и связка с адресами записи и пунктами выдачи.

**Что сделано:**

3. **API** — `GET`/`POST` `/api/contacts-content` (сохранение с правом `edit_content`); публичные данные для карты/списков через существующие эндпоинты записей и ПВЗ (с фильтром по `showOnContactsPage` где применимо).

**Основные файлы:**
- `src/app/admin/content/contacts/page.tsx`
- `src/app/api/contacts-content/route.ts`
- `src/app/(site)/contacts/page.tsx`, `src/app/(site)/contacts/local_components/ContactsMap.tsx`, `ContactRow.tsx`

- **Профиль:** форма ФИО (как в TODO); телефон — только просмотр или смена только через отдельный сценарий (если телефон = уникальный логин). Опционально: смена пароля (если есть API).

Документы в подвал: загрузка через `POST /api/upload` с полем `document` (PDF/DOC/DOCX, только `superadmin`).
- **Контент сайта** (`/admin/content/*`): доступ только у роли `superadmin` — `layout` + скрытая карточка на дашборде; POST API контента и изменение акций уже с `superadmin`; `POST /api/promotions/reorder` тоже только суперадмин.
- Юридические тексты (политика ПДн, cookies) для п.6–7 TASKS — предоставляет заказчик; в проекте — только UI, ссылки и техническая фиксация согласия.
- Все заявки на бронирование обрабатываются менеджерами вручную
- Менеджеры ведут учёт заявок НЕ на сайте
- Клиент выбирает отдел, заявка попадает в общие заявки отдела без назначенного менеджера
- Менеджеры потом берут заявки себе и присваивают их себе
